# BTC Price Telegram Bot

Небольшой Telegram-бот на Python, который в реальном времени отправляет обновления цены BTC/USDT с биржи Binance через WebSocket. Бот поддерживает подписку пользователей на поток цен с помощью команды `/start`.

>Активный бот: [@btc_price_botbot](https://t.me/btc_price_botbot)
---

## Установка

1. Клонируйте репозиторий:

```bash
git clone https://github.com/arhipov-om/btc-price-bot.git
cd btc-price-bot
```

2. Установите зависимости через pip:

```bash
python -m pip install aiogram aiohttp environs
```

Или используя `uv`:

```bash
uv sync
```

> Проект тестировался на Python >= 3.11.

3. Создайте файл `.env` в корневой папке проекта и добавьте токен вашего Telegram-бота:

```env
BOT_TOKEN=your_bot_token_here
```

> Получите токен у [@BotFather](https://t.me/BotFather) в Telegram.

## Пример запуска скрипта

```bash
python main.py
```

> Убедитесь, что файл `.env` содержит корректный `BOT_TOKEN`.

## Описание работы

- Бот подключается к WebSocket Binance (`wss://fstream.binance.com/ws/btcusdt@aggTrade`) для получения реальных цен BTC/USDT.
- Пользователи могут подписаться на обновления цен, отправив команду `/start`.
- Цены обновляются каждые 5 секунд (с троттлингом) и отправляются всем подписчикам параллельно.
- Формат сообщений: `BTC: $цена\nдд.мм.гггг, чч:мм:сс`.

## Зависимости

- `aiogram` — для работы с Telegram API.
- `aiohttp` — для подключения к WebSocket Binance.
- `environs` — для работы с переменными окружения.

## Структура кода

- `SUBSCRIBERS`: Словарь для хранения подписчиков (ID пользователя и объект сообщения).
- `SUBSCRIBERS_LOCK`: `asyncio.Lock` для безопасного доступа к словарю подписчиков.
- `price_stream`: Асинхронный генератор для получения данных о ценах через WebSocket.
- `broadcast`: Функция, рассылающая обновления цен всем подписчикам с троттлингом в 5 секунд.
- `update_message_task`: Обновляет сообщение для конкретного пользователя.
- `process_start`: Обрабатывает команду `/start`, добавляя пользователя в список подписчиков.

## Примечания

- Если сообщение не может быть отредактировано (например, пользователь заблокировал бота), ошибка обрабатывается, и бот продолжает работу.
- Для повышения надёжности добавлена обработка исключений в `broadcast` с повторной попыткой подключения через 1 секунду.
- Для масштабирования можно добавить очистку неактивных пользователей из `SUBSCRIBERS`.
